Формат входных данных:
	python [programName] [methodName] [methodArgument] [parameter option] [parameter option] ...
	python [commandName] [methodName]
[programName] - имя программы, которая запускается
[commandName] - название команды-сервиса
[methodName] - название http метода
[parameter option] - параметр команды с опциями, перечисленными через пробел

Примеры: python main.py GET http://anytask.urgu.org/ -o text.txt -m 500
         python main.py http://anytask.urgu.org/
         python help GET

Команды:
1) GET  (функционал http://anytask.urgu.org/)
    Параметры:
        1. -o filename.txt - загрузить содержимое тела ответа сервера в файл filename
        2. -m time - максимальное время ожидания от сервера в "time" s
        3. -O link - если хотим, чтобы содержимое страницы загрузилось в файл с тем же именем
                     что и на сервере

Структура проекта:
    Source - папка с исходным кодом программы
        Errors - папка с классами Exception под определенные случаи
        HTTP - папка с файлами, связанными с http: методы, их параметры и класс-обертка http запроса
        Program - основные модули, управляющие процессом: сам http-клиент, который по распарсенным
                  аргументам командной строки составляет и отправляет http запрос; парсер аргументов
                  командной строки, который оборачивает их в определенные классы
        ProgramResponses - пока заготовка, но в теории в эти классы будут оборачиваться сообщения-ответы
                           от нашей программы пользователю командной строки
        Service - хранятся классы с командами, не связанными с http, и параметры к ним
        UserRequests - хранятся классы, в которые оборачиваются запросы командной строки в зависимости
                       от самого запроса
    Tests - тесты(пока только немного для парсера)
